Simple MVP overview

One portal for alumni profiles, premium access, events and stronger SBE connection.

This first MVP focuses on a searchable alumni directory, self-managed profiles, event visibility, premium membership gating and a practical admin view for the Alumni Office. It is intentionally simple, so it can be built fast with Python, PostgreSQL and one Hetzner server.

First API endpoints

Minimal routes for Flask or FastAPI.

  • POST /auth/register for alumni sign-up with email verification.
  • POST /auth/login for session or token-based access.
  • GET /profiles with filters for year, city, sector, employer and tutorial.
  • GET /profiles/{id} for full alumni profile view.
  • PUT /me/profile for self-service profile updates.
  • GET /events and POST /events/{id}/register for event participation.
  • GET /jobs and POST /jobs for alumni career postings.
  • POST /memberships/upgrade for premium purchase and plan upgrade.
  • GET /admin/imports and POST /admin/deduplicate for office operations.

Python backend first steps

Lean stack for one senior developer and LLM-assisted coding.

  • Use FastAPI for clear APIs, automatic docs and quick iteration.
  • Use PostgreSQL as the main database from day one.
  • Use SQLAlchemy and Alembic for models and migrations.
  • Use Jinja templates if you want server-rendered HTML first, or keep this HTML as a static front-end mock while the API is built.
  • Use Celery or a simple cron job later for email reminders, event notifications and duplicate checks.
  • Use Postmark for transactional emails and verification links.
  • Use Stripe only when premium billing is ready. Until then, mark premium members manually in admin.

Deploy on Hetzner

First practical deployment path for the MVP.

  1. Create a Hetzner Cloud server, for example Ubuntu 24.04 on a CX23 or CPX21 instance.
  2. Point your domain or subdomain to the server IP in Cloudflare DNS.
  3. SSH into the server and update packages: apt update && apt upgrade -y.
  4. Install Docker and Docker Compose plugin.
  5. Run PostgreSQL and the FastAPI app in Docker containers.
  6. Use Nginx as reverse proxy on ports 80 and 443.
  7. Issue a free TLS certificate with Let's Encrypt and Certbot.
  8. Add UFW firewall rules for SSH, HTTP and HTTPS only.
  9. Set daily PostgreSQL backups to a second volume or object storage.

    environment:
      POSTGRES_DB: alumni
      POSTGRES_USER: alumni_user

    build: .
    command: uvicorn app.main:app --host 0.0.0.0 --port 8000
    depends_on:

  nginx:
    image: nginx:stable
